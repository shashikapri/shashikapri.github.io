Why This Matters

Scrum Masters often work in hybrid agile environments. Knowing other frameworks helps you collaborate, scale, and guide teams more effectively.

Agile Framework Overview

Scrum is lightweight, but XP, SAFe, and FDD offer complementary strengths.

FrameworkFocusTeam StructureKey Points
ScrumEmpirical, iterativeSingle cross‑functional teamSprints, PO, SM roles
XPTechnical excellenceSmall teamsTDD, pair programming
SAFeEnterprise scalingTeams of teamsPI Planning, ARTs
FDDFeature deliveryDomain feature teamsDomain modeling, milestones

Practical Integration

XP

  • Introduce pair programming
  • Run TDD workshops
  • Add continuous integration to DoD

SAFe

  • Support PI Planning
  • Work with RTEs
  • Facilitate ART syncs

FDD

  • Use domain modeling
  • Apply feature decomposition
  • Use milestone tracking

Decision Guide

Start ↓ Single team? → Yes → Scrum ↓ No Weak technical practices? → XP ↓ No Need enterprise coordination? → SAFe ↓ No Complex domain modeling? → FDD ↓ No Use Scrum + tailored practices

Scrum Master Competencies

  • Framework literacy
  • Context assessment
  • Hybrid implementation
  • Scaling awareness
